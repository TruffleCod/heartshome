# 心之家网页推理游戏开发说明

## 项目定位

这是一个中文网页推理游戏，灵感来自网页推理/ARG 类作品。玩家通过点击链接、阅读页面文本、输入答案来推进剧情。所有谜题和线索都藏在网页内容、页面结构、按钮文案、时间、编号、用户资料、站内信、咨询记录等细节中。

游戏暂定名：《心之家》。

## 技术方向

使用 Vite + React 开发。
优先做成前端静态项目。
暂时不需要真实后端、真实数据库、真实登录系统。
玩家进度可以先用 localStorage 保存。
剧情文本、帖子、账号、站内信、咨询记录等内容尽量放在 src/data/*.json 中，方便后续修改。

## 交互原则

游戏只需要两类基础操作：
1. 点击链接、按钮、页面元素。
2. 输入文本答案、账号、密码或日期。

不要设计复杂操作，不要依赖拖拽、战斗、实时反应、三维场景。

## 视觉风格

第一层是可信的中文心理互助论坛。
风格应当干净、普通、略微陈旧、可信，不要一开始就恐怖化。
恐怖感来自后续文本错位、页面状态变化、重复、删词、语气变冷，而不是跳脸或夸张血腥视觉。

## 数据规划

建议创建：

- src/data/posts.json
- src/data/accounts.json
- src/data/messages.json
- src/data/counselingRecords.json
- src/data/assessmentQuestions.json

## 开发原则

先实现页面结构和跳转，再逐步加入正式剧情文本和谜题。
不要擅自改动核心剧情设定。
不要擅自更改文案。根据开发者提供的截图，准确识别其中的文字，并放入网页上。
不要主动加入大型依赖或复杂后端。
每次修改后尽量确保 npm run dev 可以正常运行。
